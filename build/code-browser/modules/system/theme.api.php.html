  12.  12  * Inside Drupal, the theme layer is utilized by the use of the theme()
  13.  13  * function, which is passed the name of a component (the theme hook)

  17.  17  * hooks, which can be used to provide 'fallback' implementations to
  18.  18  * allow for more specific control of output. For example, the function:
  19.  19  * theme(array('table__foo', 'table'), $variables) would look to see if
  20.  20  * 'table__foo' is registered anywhere; if it is not, it would 'fall back'
  21.  21  * to the generic 'table' implementation. This can be used to attach specific
  22.  22  * theme functions to named objects, allowing the themer more control over
  23.  23  * specific types of output.

  26.  26  * module that owns it, so that Drupal can tell what to do with it and
  27.  27  * to make it simple for themes to identify and override the behavior
  28.  28  * for these calls.
  29.  29  *

  31.  31  * array of arrays with information about the hook. It describes the
  32.  32  * arguments the function or template will need, and provides
  33.  33  * defaults for the template in case they are not filled in. If the default
  34.  34  * implementation is a function, by convention it is named theme_HOOK().
  35.  35  *
  36.  36  * Each module should provide a default implementation for theme_hooks that
  37.  37  * it registers. This implementation may be either a function or a template;
  38.  38  * if it is a function it must be specified via hook_theme(). By convention,
  39.  39  * default implementations of theme hooks are named theme_HOOK. Default
  40.  40  * template implementations are stored in the module directory.
  41.  41  *
  42.  42  * Drupal's default template renderer is a simple PHP parsing engine that
  43.  43  * includes the template and stores the output. Drupal's theme engines
  44.  44  * can provide alternate template engines, such as XTemplate, Smarty and
  45.  45  * PHPTal. The most common template engine is PHPTemplate (included with
  46.  46  * Drupal and implemented in phptemplate.engine, which uses Drupal's default
  47.  47  * template renderer.
  48.  48  *
  49.  49  * In order to create theme-specific implementations of these hooks, themes can

  51.  51  * These implementations will be used instead of the default implementation. If
  52.  52  * using a pure .theme without an engine, the .theme is required to implement
  53.  53  * its own version of hook_theme() to tell Drupal what it is implementing;
  54.  54  * themes utilizing an engine will have their well-named theming functions
  55.  55  * automatically registered for them. While this can vary based upon the theme
  56.  56  * engine, the standard set by phptemplate is that theme functions should be
  57.  57  * named THEMENAME_HOOK. For example, for Drupal's default theme (Bartik) to

  59.  59  * bartik_table().
  60.  60  *
  61.  61  * The theme system is described and defined in theme.inc.
  62.  62  *
  63.  63  * @see theme()
  64.  64  * @see hook_theme()
