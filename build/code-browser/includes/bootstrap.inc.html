  265.  265  * directly. However since DrupalCacheArray holds partial content by design, it

  269.  269  * write directly to the contents of nested arrays contained in this object.
  270.  270  * Only writes to the top-level array elements are possible. So if you
  271.  271  * previously had set $object['foo'] = array(1, 2, 'bar' => 'baz'), but later

  273.  273  * a targeted write like $object['foo']['bar'] = 'foobar'. Instead, you must
  274.  274  * overwrite the entire top-level 'foo' array with the entire set of new
