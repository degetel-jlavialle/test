  14. 14  * Explain your records in the {node_access} table.
  15. 15  *
  16. 16  * In order to help developers and administrators understand the forces
  17. 17  * that control access to any given node, the DNA module provides the
  18. 18  * Devel Node Access block, which lists all the grant records in the
  19. 19  * {node_access} table for that node.
  20. 20  *
  21. 21  * However, every Node Access module is free in how it defines and uses the
  22. 22  * 'realm' and 'gid' fields in its records in the {node_access} table, and
  23. 23  * it's often difficult to interpret them. This hook passes each record
  24. 24  * that DNA wants to display, and the owning module is expected to return
  25. 25  * an explanation of that record.
  26. 26  *

  28. 28  * that administrators seeking help can present English explanations.
  29. 29  *
  30. 30  * @param $row
  31. 31  *   The record from the {node_access} table, as object. The member fields are:
  32. 32  *   nid, gid, realm, grant_view, grant_update, grant_delete.
  33. 33  *
  34. 34  * @return
  35. 35  *   A string with a (short!) explanation of the given {node_access} row,
  36. 36  *   to be displayed in DNA's 'Devel Node Access' block. It will be displayed
  37. 37  *   as HTML; any variable parts must already be sanitized.
  38. 38  *
  39. 39  * @see hook_node_access_records()
