  26.  26 OVERVIEW
  27.  27 ------------------------------------------------------------------------------
  28.  28 To put it simply, Krumo is a replacement for print_r() and var_dump(). By 
  29.  29 definition Krumo is a debugging tool (for PHP5), which displays structured 
  30.  30 information about any PHP variable.
  31.  31 
  32.  32 A lot of developers use print_r() and var_dump() in the means of debugging 
  33.  33 tools. Although they were intended to present human readble information about a 
  34.  34 variable, we can all agree that in general they are not. Krumo is an 
  35.  35 alternative: it does the same job, but it presents the information beautified 
  36.  36 using CSS and DHTML. 
  37.  37 
  38.  38 EXAMPLES
  39.  39 ------------------------------------------------------------------------------
  40.  40 Here's a basic example, which will return a report on the array variable passed 
  41.  41 as argument to it:
  42.  42 
  43.  43  krumo(array('a1'=&gt; 'A1', 3, 'red'));
  44.  44 
  45.  45 You can dump simultaneously more then one variable - here's another example:
  46.  46 
  47.  47  krumo($_SERVER, $_REQUEST);
  48.  48 
  49.  49 You probably saw from the examples above that some of the nodes are expandable, 
  50.  50 so if you want to inspect the nested information, click on them and they will 
  51.  51 expand; if you do not need that information shown simply click again on it to 
  52.  52 collapse it. Here's an example to test this:
  53.  53 
  54.  54  $x1-&gt;x2-&gt;x3-&gt;x4-&gt;x5-&gt;x6-&gt;x7-&gt;x8-&gt;x9 = 'X10';
  55.  55  krumo($x1);
  56.  56 
  57.  57 The krumo() is the only standalone function from the package, and this is 
  58.  58 because basic dumps about variables (like print_r() or var_dump()) are the most 
  59.  59 common tasks such functionality is used for. The rest of the functionality can 
  60.  60 be called using static calls to the Krumo class. Here are several more examples:
  61.  61 
  62.  62  // print a debug backgrace
  63.  63  krumo::backtrace();
  64.  64 
  65.  65  // print all the included(or required) files
  66.  66  krumo::includes();
  67.  67  
  68.  68  // print all the included functions
  69.  69  krumo::functions();
  70.  70  
  71.  71  // print all the declared classes
  72.  72  krumo::classes();
  73.  73  
  74.  74  // print all the defined constants
  75.  75  krumo::defines();
  76.  76 
  77.  77  ... and so on, etc.

  91.  91 SKINS
  92.  92 ------------------------------------------------------------------------------
  93.  93 There are several skins pre-installed with this package, but if you wish you can 
  94.  94 create skins of your own. The skins are simply CSS files that are prepended to 
  95.  95 the result that Krumo prints. If you want to use images in your CSS (for 
  96.  96 background, list-style, etc), you have to put &quot;%URL%&quot; in front of the image URL 
  97.  97 in order hook it up to the skin folder and make the image web-accessible.
  98.  98 
  99.  99 Here's an example:
  100. 100 
  101. 101  ul.krumo-first {background: url(%url%bg.gif);}
