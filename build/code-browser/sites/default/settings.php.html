  57.  57  *

  59.  59  * connections that Drupal may use.  Drupal is able to connect
  60.  60  * to multiple databases, including multiple types of databases,
  61.  61  * during the same request.
  62.  62  *
  63.  63  * Each database connection is specified as an array of settings,
  64.  64  * similar to the following:
  65.  65  * @code
  66.  66  * array(
  67.  67  *   'driver' => 'mysql',
  68.  68  *   'database' => 'databasename',
  69.  69  *   'username' => 'username',
  70.  70  *   'password' => 'password',

  72.  72  *   'port' => 3306,
  73.  73  *   'prefix' => 'myprefix_',
  74.  74  *   'collation' => 'utf8_general_ci',

  76.  76  * @endcode
  77.  77  *
  78.  78  * The "driver" property indicates what Drupal database driver the
  79.  79  * connection should use.  This is usually the same as the name of the

  87.  87  * transaction support for a given database, set the 'transactions' key
  88.  88  * to TRUE.  To disable it, set it to FALSE.  Note that the default value
  89.  89  * varies by driver.  For MySQL, the default is FALSE since MyISAM tables

  93.  93  * A target database allows Drupal to try to send certain queries to a

  96.  96  * to a slave server when appropriate and if one is not available will simply
  97.  97  * fall back to the single master server.

  100. 100  * @code
  101. 101  * $databases['default']['default'] = $info_array;
  102. 102  * $databases['default']['slave'][] = $info_array;

  109. 109  * (the second level default).  The second and third lines create an array
  110. 110  * of potential slave databases.  Drupal will select one at random for a given
  111. 111  * request as needed.  The fourth line creates a new database with a name of
  112. 112  * "extra".
